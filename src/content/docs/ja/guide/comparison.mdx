# Project Comparison

Aether Lens は、開発環境における「個人の生産性」と「高速なフィードバック」に特化したツールです。ここでは、Kubernetes ネイティブなテストオーケストレーターとして知られる **Testkube** などの他ツールとの違いを整理します。

## 比較の核心: Personal vs. Team

| 特徴 | Aether Lens | Testkube / 他の CI ツール |
| :--- | :--- | :--- |
| **主要ターゲット** | **個人 (Personal)** | **チーム / 組織 (Team)** |
| **主な用途** | **Local Dev Loop の加速** | テストオーケストレーション |
| **フィードバック速度** | **ミリ秒〜秒単位 (即時)** | 分単位 (パイプライン依存) |
| **解析エンジン** | Vibe Test Insight (AI 意図解析) | 定義済みテストスイート |
| **インフラ** | 開発者の手元 + Aether Platform | Kubernetes クラスター内 |
| **実行タイミング** | 保存のたびに自動 (Live) | Push/PR ごとのトリガー |

## Aether Lens が選ばれる理由

### 1. 「気分」を読み取るフィードバック
Testkube は記述されたテストを効率的に動かすことに長けていますが、Aether Lens は「今何を確認すべきか」を AI が提案します。バイブコーディングにおいて、開発者の思考を止めないことが最大の価値です。

### 2. Personal Local Dev Loop への特化
多くの CI/CD ツールは「ビルド・デプロイ」をゴールとしますが、Aether Lens は「コードを書いている最中」の試行錯誤を支えます。プラットフォーム上の Headless Browser をあたかも手元のツールのように操る体験は Aether Lens 独自のものです。

### 3. Aether Platform 搭載の優位性
Aether Platform 上では、追加のセットアップなしに Kubernetes 上の検証リソースを動的に活用できます。これは、個人の開発速度を保ちつつ、チームが求める品質基準（VRT 等）をクリアするための最短ルートです。

## 使い分けのガイドライン

- **Aether Lens を使うべき時**:
  - 新機能の実装中に、即座に UI の崩れやロジックの影響を確認したい。
  - AI エージェント（Cursor, Claude 等）と対話しながら、自律的にコードを修正・検証させたい。
  - ローカル PC のリソースを消費せずに、高度な検証を行いたい。

- **Testkube / CI ツールを使うべき時**:
  - チーム全体の品質ゲートとして、全テストを網羅的に走らせたい。
  - 本番同様の Kubernetes クラスター内で長期的なテストレポートを蓄積したい。
  - 複数のマイクロサービスにまたがる大規模な回帰テストを定期実行したい。
