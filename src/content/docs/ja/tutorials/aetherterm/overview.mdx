## 概要

AetherTermは単なるコマンドラインではありません。あなたのインフラストラクチャを理解するインテリジェントなパートナー（エージェント）です。自然言語の意図を正確なシェルコマンドに翻訳し、複雑な構文を覚える必要性を最小限に抑えます。

## 技術的インテリジェンス

### コンテキスト認識AI

AetherTermは、クラスタの状態とドキュメントのコンテキストに統合されます。

- **モデル**: DevOpsおよびKubernetes運用に特化したファインチューニングされたLLMを搭載。
- **安全性**: 「Human-in-the-loop」設計により、あなたの明示的で意味のある承認なしにコマンドが実行されることはありません。
- **コンテキスト注入**: シェルは、現在のディレクトリ、アクティブなKubernetesネームスペース、および最近のエラーログを認識しています。

### 安全な実行

- **サンドボックス化**: コマンドは制限された機能を持つシェルで実行されます。
- **監査ログ**: AIが生成したすべてのコマンドとその出力は、コンプライアンスとトレーニングのために記録されます。

## Model Context Protocol (MCP) 統合

AetherTermは**Model Context Protocol (MCP)**を実装しており、ファイアウォールの背後にあるものを含め、既存のツールやデータソースと安全にインターフェースすることができます。

### オンプレミスツール統合

複雑なAPIグルーコードなしで、AetherTermを社内のセキュリティツールに接続します。

- **シナリオ**: レガシーなオンプレミスセキュリティスキャナーとの統合。
- **ワークフロー**:
  1. AetherTermは、ネットワーク接続サービスを介してオンプレミススキャナーに接続します。
  2. エージェントは、カスタマイズされたビルドアーティファクトに対してスキャンをトリガーします。
  3. エージェントは独自のレポート形式を解析し、脆弱性をターミナルの概要に直接ハイライト表示します。

### 手動トイルの自動化 (IaC)

何時間もかかる手動の構成監査を、インテリジェントなエージェントアクションに置き換えます。

- **問題**: 新しいコンプライアンス基準に合わせて、数百のTerraformリソース定義とHelm値を手動で更新する。
- **ソリューション**:
  - エージェントはMCPを使用してIaCリポジトリをクロールし、現在の状態を理解します。
  - 非推奨のAPIバージョンや安全でない構成（例：公開S3バケット）を特定します。
  - **自己修正**: 修正されたTerraformコードを含むプルリクエストを自動的にオープンします。
  - _結果_: 人間が3日かけて行っていた「コピペ」作業が10分で完了します。

## ユーザーストーリー: 午前3時の消火活動

> **シナリオ**: 「本番環境で重要なマイクロサービスがクラッシュループしており、オンコールエンジニアは迅速に診断する必要がある。」

**課題:**
エンジニアは*何を*したいか（クラッシュしているポッドのログを確認する）はわかっているが、特定のデプロイメントハッシュに対する正確な `kubectl` セレクタ構文を忘れてしまった。

**AetherTermソリューション:**

1. エンニアは次のように入力します: _"Why is the payment-service restarting?" (なぜ決済サービスが再起動しているの？)_
2. AetherTermはクライストイベントを分析し、次を提案します:
   ```bash
   kubectl get events --sort-by='.lastTimestamp' --field-selector involvedObject.name=payment-service-7d8b9c
   ```
3. エンジニアは **Enter** を押します。結果は `OOMKilled` を示します。
4. エンジニアは次のように入力します: _"Show me the previous logs for the crashed instance." (クラッシュしたインスタンスの以前のログを見せて)_
5. AetherTermは次を提案します:
   ```bash
   kubectl logs -p -l app=payment-service --tail=50
   ```
6. 問題が特定され（新機能のメモリリーク）、数分でロールバックされました。
